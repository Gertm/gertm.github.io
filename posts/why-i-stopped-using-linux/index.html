Why I stopped using Linux

Date [ Oct 15, 2016 ] Tags [ linux windows ]

TL;DR: Updating video drivers screwed my system once too many. Reinstalled to Windows and never looked back. Now the PC works for me, instead of me working to keep it running.

As most people of my age, I started out on my parent’s computer. One of the first ones I had access to was a Philips MSX. Quickly moved on from that one to a 8088, which I used for a couple of years. This is where I learned DOS and where I got my love for command line interfaces. Since my first 5 years of computer usage was command line only, it still feels like home to this day. Windows came along and I started using the fancy 3.0 newness. I did use Windows 1.0 on the 8088, but didn’t find it that interesting at the time.

In 2002 I switched to Linux as my full time OS at home. Before that I had spent about 3 years getting familiar with how Linux worked through a dual boot system. Slackware at first, jumping over to Gentoo when that became available. Tweaking my system was almost a daily activity. There was always something to adjust or try out and I was enjoying it! I even played all my games on Linux, which worked a lot better than on Windows on the same PC.

Then I started work. Work was Windows/.NET-oriented, as most businesses are in Belgium. After a year or two, I found a way to run Linux on my work laptop, using a virtual machine for when I absolutely needed Windows. Now I was tweaking and tinkering with my work laptop as well as my home machines. A couple of years later I got the opportunity to replace the Windows servers at work with Linux servers. That’s probably the time when I learned the most about system administration and keeping production systems running.

Work changed again, this time towards being a full time programmer. There was little time left for system administration, so this got put on the backburner. (You can already see where this is headed, right?) I started spending most of my days in Visual Studio after I changed jobs, to a real programming job. I still ran Linux, with a Windows VM for all of my programming. Windows 7 wasn’t cutting it for me as a day to day OS, so I ran it on top of a Fedora in a VirtualBox VM. This was fine for a year or 3. The dev environment was portable, which came in handy a couple of times. Plus it’s easy to make a backup.

When the time came to replace the first laptop, Windows 10 was just around the corner. I decided to wait with reinstalling the Windows 8 on the laptop until 10 was available and try it out. Windows 10 was actually much nicer than I had hoped it would be. Windows stayed on my work pc, while I used Fedora at home. This situation was fine for quite some time. But after one last failed video driver update that threw me back to init 3 at boot, right at a time when I really needed my pc to work, I decided that enough is enough.

Got a Windows license through work and reinstalled my PC with Windows 10. It’s what I was using at work anyway so I was already comfortable with it. Haven’t really looked back since. Everything works well, and it’s enjoyable to work with the pc again. I even found some tools that make the transition for a former Linux user a bit easier. Especially Chocolatey and WSL are a big help. Also, it seems to me that the days of constant malware threats are over. I haven’t had any problems yet (that I know of, of course). Smart online behavior helps a lot of course. That’s something I’ve had to re-learn coming from Linux.

I’m not saying I’ve cut all the Linux out of my life. My VPS still runs Linux and will continue to do so in the future. I’m looking forward to start developing for Linux again even.
